# Стеллажный склад

Фронтенд: `WarehousePage.jsx` (`/admin/warehouse`).

API: `/api/warehouse` — `backend/src/routes/warehouse.js`.

## Иерархия

```
Склад (warehouses_s) → Стеллаж (racks_s) → Полка (shelves_s)
                                                ├── Россыпь (shelf_items_s)
                                                └── Коробки (shelf_boxes_s)
                                                     └── Товары (shelf_box_items_s)
```

## Типы складов (warehouse_type)

| Тип | Описание |
|---|---|
| `fbs` | Стеллажный склад (FBS) |
| `fbo` | Паллетный склад |
| `both` | Комбинированный |
| `visual` | Визуальный стеллажный склад |
| `visual_pallet` | Визуальный паллетный склад |

## Таблицы

### warehouses_s
name, external_id, warehouse_type, active, notes.

### racks_s
warehouse_id (FK), name, number, code, barcode_value (UNIQUE).

### shelves_s
rack_id (FK), name, number, code, barcode_value (UNIQUE), uses_boxes (BOOLEAN).

### shelf_items_s
shelf_id (FK), product_id (FK), quantity. UNIQUE(shelf_id, product_id).

### shelf_boxes_s
shelf_id (FK), position, name, barcode_value (UNIQUE), product_id, task_id, quantity, box_size, status (default 'closed'), confirmed.

### shelf_box_items_s
shelf_box_id (FK), product_id (FK), quantity. UNIQUE(shelf_box_id, product_id).

## Штрихкоды

- Стеллажи: `racks_s.barcode_value` (генерируется: 100000 + id)
- Полки: `shelves_s.barcode_value` (генерируется: 200000000 + id)
- Коробки на полках: `shelf_boxes_s.barcode_value`

Сотрудник сканирует штрихкод полки для начала инвентаризации или перемещения.

## Режимы полки

- `uses_boxes = false` — товары россыпью (`shelf_items_s`)
- `uses_boxes = true` — товары в коробках (`shelf_boxes_s` → `shelf_box_items_s`)

## Визуальный склад

Графическое представление структуры: стеллажи и полки в виде сетки.
- FBS: компонент `FBSVisualView`, API `GET /api/warehouse/visual-fbs/:warehouseId`
- Перемещение: `POST /api/warehouse/visual-fbs/move`

## Аудит (shelf_movements_s)

Все операции логируются:

| operation_type | Описание |
|---|---|
| `inventory` | Результат инвентаризации |
| `correction` | Ручная корректировка |
| `stock_in` | Приход |
| `stock_out` | Расход |

Поля: shelf_id, product_id, quantity_before, quantity_after, quantity_delta, user_id, task_id, source, notes.
