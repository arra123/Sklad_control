# Паллетный склад

Фронтенд: `FBOPage.jsx` (`/admin/fbo`).

API: `/api/fbo` — `backend/src/routes/fbo.js`.

## Иерархия

```
Склад (warehouses_s, type='fbo') → Ряд (pallet_rows_s) → Паллет (pallets_s)
                                                              ├── Россыпь (pallet_items_s)
                                                              └── Коробки (boxes_s)
                                                                   └── Товары (box_items_s)
```

## Таблицы

### pallet_rows_s
warehouse_id (FK), number, name. UNIQUE(warehouse_id, number).

### pallets_s
row_id (FK), number, name, barcode_value (UNIQUE), uses_boxes (BOOLEAN, default true). UNIQUE(row_id, number).

Имя паллета генерируется автоматически: `Р{row.number}П{pallet.number}`.

### boxes_s
barcode_value (UNIQUE), product_id, pallet_id (FK), task_id, quantity, box_size (default 50), status (open/closed), confirmed, is_remainder, remainder_shelf_id, closed_at.

### box_items_s
box_id (FK), product_id (FK), quantity. UNIQUE(box_id, product_id). Мульти-продуктовое содержимое.

### pallet_items_s
pallet_id (FK), product_id (FK), quantity. UNIQUE(pallet_id, product_id). Товары россыпью на паллете.

## Режимы паллета

- `uses_boxes = true` — товар хранится в коробках (`boxes_s` → `box_items_s`)
- `uses_boxes = false` — товар россыпью (`pallet_items_s`)

Нельзя смешивать: если есть коробки, нельзя добавить россыпь (и наоборот) без очистки.

## Коробки (boxes_s)

| Статус | Описание |
|---|---|
| `open` | Коробка открыта, можно добавлять товары |
| `closed` | Коробка закрыта и подтверждена |

- `box_size` — максимальное кол-во товаров в коробке
- `is_remainder` — неполная коробка (остаток от упаковки)
- `confirmed` — подтверждена после упаковки
- `remainder_shelf_id` — если остаток перенесён на FBS полку

## Визуальный склад

Компонент `FBOVisualView` — графическое отображение рядов и паллетов с коробками.
- `GET /api/fbo/visual/:warehouseId` — данные для визуализации
- `POST /api/fbo/visual/move` — перемещение коробки между паллетами

## Перемещения

Добавление/изменение товаров россыпью логируется в `movements_s`:
- `external_to_pallet` — добавление
- `pallet_correction_in` / `pallet_correction_out` — ручное редактирование
