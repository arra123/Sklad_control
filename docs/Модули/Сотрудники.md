# Сотрудники

Фронтенд: `StaffPage.jsx` (`/admin/staff`).

API: `/api/staff` — `backend/src/routes/staff.js`.

Доступ: только `admin` (permission: `staff.view`, `staff.edit`).

## Сущности

### employees_s — Сотрудники

| Поле | Тип | Описание |
|---|---|---|
| full_name | VARCHAR(255) | ФИО |
| position | VARCHAR(255) | Должность |
| phone | VARCHAR(50) | Телефон |
| department | VARCHAR(255) | Отдел |
| active | BOOLEAN | Активен |
| gra_balance | NUMERIC(18,3) | Баланс [[GRACoin]] |
| external_employee_id | INTEGER | ID во внешней БД (UNIQUE) |

### users_s — Учётные записи

| Поле | Тип | Описание |
|---|---|---|
| username | VARCHAR(100) | Логин (UNIQUE) |
| password_hash | VARCHAR(255) | Хеш пароля (bcryptjs) |
| password_plain | VARCHAR(255) | Пароль в открытом виде (для выдачи) |
| role | VARCHAR(20) | admin / manager / employee |
| role_id | INTEGER | FK → roles_s |
| employee_id | INTEGER | FK → employees_s |
| active | BOOLEAN | Активен |

### roles_s — Роли

| Поле | Тип | Описание |
|---|---|---|
| name | VARCHAR(100) | Название (UNIQUE) |
| permissions | JSONB | Массив строк-разрешений |

Предустановленные роли:

| Роль | Права |
|---|---|
| Администратор | dashboard, products.view, products.edit, warehouse.view, warehouse.edit, tasks.view, tasks.create, tasks.execute, staff.view, staff.edit, movements.view, movements.edit, settings, analytics, errors, roles.manage |
| Менеджер | dashboard, products.view, warehouse.view, tasks.view, tasks.create, staff.view, movements.view, analytics, errors |
| Сотрудник | tasks.execute, movements.edit |

## Внешняя БД сотрудников

Система может подтягивать сотрудников из внешней PostgreSQL базы (сайт сборки).

Конфигурация в `backend/src/config.js`:
```js
externalDb: {
  host: process.env.EXT_DB_HOST,
  port: process.env.EXT_DB_PORT,
  database: process.env.EXT_DB_NAME,
  user: process.env.EXT_DB_USER,
  password: process.env.EXT_DB_PASSWORD,
}
```

Эндпоинт: `GET /api/staff/external-employees` — список сотрудников из внешней БД (кроме уже привязанных).

При создании сотрудника можно указать `external_employee_id` для привязки.

## Создание сотрудника

`POST /api/staff/employees` — создаёт сотрудника и опционально учётную запись (username, password, role, role_id).

## API эндпоинты

### Сотрудники
| Метод | Путь | Описание |
|---|---|---|
| GET | /employees | Список сотрудников с пользователями |
| GET | /external-employees | Внешние сотрудники (для привязки) |
| POST | /employees | Создать сотрудника (+учётная запись) |
| GET | /employees/:id/credentials | Учётные данные (логин/пароль) |
| PUT | /employees/:id | Обновить сотрудника |
| DELETE | /employees/:id | Удалить сотрудника |

### Роли
| Метод | Путь | Описание |
|---|---|---|
| GET | /roles | Список ролей |
| POST | /roles | Создать роль |
| PUT | /roles/:id | Обновить роль |
| DELETE | /roles/:id | Удалить роль |

### Пользователи
| Метод | Путь | Описание |
|---|---|---|
| GET | /users | Список пользователей |
| POST | /users | Создать учётную запись |
| PUT | /users/:id | Обновить пользователя |
| DELETE | /users/:id | Удалить пользователя |
