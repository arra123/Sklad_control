# Задачи

## Типы задач

| task_type | Описание |
|---|---|
| `inventory` | Инвентаризация — пересчёт товаров |
| `packaging` | Упаковка — фасовка в коробки |
| `production_transfer` | Передача на производство |

## Статусы

```
new → in_progress → completed
                  → cancelled
```

## Инвентаризация (inventory)

1. Админ создаёт задачу, выбирает полку/полки
2. Сотрудник начинает задачу (`POST /tasks/:id/start`)
3. Сканирует штрихкод полки для привязки
4. Сканирует товары (`POST /tasks/:id/scan`)
5. При несовпадении — `POST /tasks/:id/report-error`
6. Мульти-полка: `POST /tasks/:id/next-shelf`
7. Завершение: `POST /tasks/:id/complete`

### Результат инвентаризации
- Сканирования записываются в `inventory_task_scans_s`
- Расхождения видны в аналитике
- Движения логируются в `shelf_movements_s`

## Аналитика

- **Свежесть**: настройки `inventory_fresh_hours` (24ч) и `inventory_stale_hours` (72ч)
- **Цвета**: green (свежий) → amber (устаревающий) → red (просроченный)
- **Отчёты**: аудит, табличный, обзор, история
