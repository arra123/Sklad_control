# Аналитика

Фронтенд: `AnalyticsPage.jsx` (`/admin/analytics`).

Доступ: только `admin` (permission: `analytics`).

## Версии

Страница аналитики поддерживает два представления:

| Версия | Компонент | Описание |
|---|---|---|
| V1 (Карточки) | `InventoryAnalyticsView` | Карточки с обзором |
| V2 (Дерево) | `InventoryAnalyticsV2` | Дерево: склад → стеллаж → полка, табличный отчёт |

Переключение через toggle в интерфейсе. По умолчанию V2.

## API эндпоинты аналитики

Все в `/api/tasks/analytics/*`:

| Эндпоинт | Описание |
|---|---|
| `GET /analytics/summary` | Общая аналитика: кол-во задач, средние показатели |
| `GET /analytics/inventory-overview` | Обзор инвентаризации: статус полок, свежесть |
| `GET /analytics/inventory-history` | История: хронология инвентаризаций |
| `GET /analytics/audit-report` | Аудит-отчёт: расхождения, ошибки |
| `GET /analytics/table-report` | Табличный отчёт: дерево склад → стеллаж → полка |
| `GET /stats/summary` | Сводка: количество задач по статусам |
| `GET /:id/analytics` | Аналитика конкретной задачи |

## Свежесть инвентаризации

Система отслеживает, когда последний раз инвентаризировалась каждая полка/коробка. Цветовая индикация:

| Состояние | Условие | Цвет |
|---|---|---|
| Свежий | < `inventory_fresh_hours` | Зелёный |
| Устаревающий | между fresh и stale | Жёлтый |
| Просроченный | > `inventory_stale_hours` | Красный |

Настройки (из [[Настройки]]):
- `inventory_fresh_hours` — порог свежести (default: 24 часа)
- `inventory_stale_hours` — порог просрочки (default: 72 часа)
- `inventory_color_fresh` — цвет свежего
- `inventory_color_warn` — цвет предупреждения
- `inventory_color_stale` — цвет просроченного

## Дерево (V2)

Иерархическое представление: Склад → Стеллаж → Полка.
- Для каждого узла: дата последней инвентаризации, статус свежести
- Кликабельные коробки и полки
- Подробные подписи с количеством товаров
- Цветные склады (10 цветовых тем)

Данные: `GET /api/tasks/analytics/table-report`.
