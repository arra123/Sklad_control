## v7.52.0

### Товары · один ШК на два товара (точечное исключение)

- **ШК `400000000282` теперь на обоих товарах** — старом (id 28) и новом (id 192): для нового ещё нет этикеток, оба временно проводятся по одной. Добавляется автоматически при старте сервера и восстанавливается после импорта из МойСклад (импорт перезаписывает `barcode_list`).
- **Это исключение, а не правило.** Список исключений — `backend/src/utils/sharedBarcodes.js`: делить ШК могут только явно перечисленные там товары. Для всех остальных теперь наоборот строже: при **добавлении** ШК в карточке появилась глобальная проверка «Штрих-код уже принадлежит другому товару» (раньше она была только при переименовании).
- **Сканирование с shared-ШК стало умным.** Если ШК принадлежит двум товарам, система выбирает товар по контексту: в инвентаризации — тот, что числится на полке/в коробке задачи; в сборке комплекта — тот, что входит в состав комплекта; без контекста — детерминированно первый (меньший id, т.е. старый товар).
- Файлы: `backend/src/utils/sharedBarcodes.js` (новый), `backend/src/db/schema.js`, `backend/src/utils/catalogImport.js`, `backend/src/routes/products.js`, `backend/src/routes/tasks.js`, `backend/src/routes/assembly.js`, `backend/src/routes/packing.js`, `backend/src/routes/movements.js`.

## v7.50.0

### Заказы · черновики со ссылкой + ручное добавление товара в заказ

- **Кнопка «Сохранить черновик».** Во вкладке «Вручную» можно завести заказ, даже если не всё выбрано/заполнено — статус `Черновик`, заказ сразу попадает в список. Черновик можно создать и **пустым**, а состав добавить позже.

Пересчитываются баночки и сумма, собранное подрезается под новые количества (`PUT /orders/:id/items`). Решает кейс, когда AI не распознал позицию (напр. NAC 1200 мг) — теперь её просто дособрать вручную, без товаров-пустышек.
