## CI/CD Pipeline

Файл: `.github/workflows/deploy.yml`

Триггер: `push` в ветку `main`.

```mermaid
graph TD
    Push[git push main] --> Checkout[Checkout code]
    Checkout --> Setup[Setup Node.js 20]
    Setup --> Install[npm ci frontend]
    Install --> Build["npx vite build --mode sklad"]
    Build --> Clean[SSH: rm old assets]
    Clean --> Upload1[SCP: upload backend]
    Clean --> Upload2[SCP: upload frontend/dist]
    Upload1 --> Env[SSH: ensure .env keys]
    Upload2 --> Env
    Env --> Deps["SSH: npm install --production"]
    Deps --> Restart["SSH: pm2 delete c-site, fuser -k 3017, pm2 start src/server.js --name c-site, pm2 save"]
```

### Шаги деплоя

1. **Checkout** — клонирование репозитория
2. **Setup Node.js 20** — с кэшем `frontend/package-lock.json`
3. **Install** — `cd frontend && npm ci`
4. **Build** — `npx vite build --mode sklad`
5. **Clean** — SSH: удаление старых ассетов `frontend/dist/assets/*`
6. **Upload backend** — SCP: `backend/src/`, `backend/package.json`, `backend/package-lock.json`
7. **Upload frontend** — SCP: `frontend/dist/`
8. **Ensure .env** — SSH: добавление OZON ключей если отсутствуют
9. **Install deps** — SSH: `npm install --production`
10. **Restart** — SSH: `pm2 delete c-site`, `fuser -k 3017`, `pm2 start src/server.js --name c-site`, `pm2 save`

## Переменные окружения (`backend/.env`)

| Переменная | Описание |
|---|---|
| `PORT` | Порт сервера (default: 3020) |
| `APP_BASE_PATH` | Базовый путь (например `/sklad`) |
| `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD` | Подключение к PostgreSQL |
| `JWT_SECRET` | Секрет для JWT |
| `JWT_EXPIRES_IN` | Срок жизни токена (default: 7d) |
| `MOYSKLAD_TOKEN` | Токен API МойСклад |
| `WB_TOKEN` | Токен Wildberries Content API |
| `OZON_CLIENT_ID`, `OZON_API_KEY` | OZON магазин 1 |
| `OZON2_CLIENT_ID`, `OZON2_API_KEY` | OZON магазин 2 |
| `EXT_DB_HOST`, `EXT_DB_PORT`, `EXT_DB_NAME`, `EXT_DB_USER`, `EXT_DB_PASSWORD` | Внешняя БД сотрудников |
| `CATALOG_SOURCE_DIR` | Путь к экспортам МойСклад |
| `SEED_ADMIN_LOGIN`, `SEED_ADMIN_PASSWORD` | Начальный логин/пароль admin |

## Версионирование

Версия хранится в `frontend/src/components/layout/AdminLayout.jsx` (~строка 228):
```jsx
<p className="text-[10px] ...">v2.9.0</p>
```

При каждом обновлении инкрементировать:
- **patch** (+0.0.1) — баг-фиксы
- **minor** (+0.1.0) — новые фичи
- **major** (+1.0.0) — крупные изменения
