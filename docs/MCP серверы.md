# MCP серверы

> Model Context Protocol — серверы, расширяющие возможности Claude Code.
> Установлены для проекта `C:\ARRA\Sklad_control`.

## Установленные серверы (13 шт.)

### 1. windows-mcp
**Что делает:** Полный контроль над Windows десктопом — клики, скриншоты, ввод текста, скролл, управление окнами, процессами, реестром, уведомления.

**Когда использовать:**
- Нужно проверить как выглядит приложение на экране — сделать скриншот
- Автоматизировать рутину: открыть программу, нажать кнопки, заполнить формы
- Работа с приложениями, у которых нет CLI/API (только GUI)
- Мониторинг — периодические скриншоты для проверки состояния

**Пример запроса:** *«Открой браузер, зайди на сайт склада и сделай скриншот главной страницы»*

---

### 2. github
**Что делает:** Работа с GitHub через API — Pull Requests, Issues, репозитории, ветки, коммиты, код-ревью, релизы, форки.

**Когда использовать:**
- Создать/посмотреть/прокомментировать Pull Request
- Управление Issues — создать баг, закрыть задачу, добавить лейбл
- Посмотреть историю коммитов в другой ветке
- Сравнить ветки, проверить CI/CD статус
- Code review — прокомментировать строки в PR

**Пример запроса:** *«Создай issue: баг в инвентаризации — не сохраняется количество»*

---

### 3. filesystem
**Что делает:** Прямой доступ к файловой системе — чтение, запись, создание, удаление, перемещение файлов и папок. Получение метаданных (размер, дата изменения).

**Когда использовать:**
- Массовые операции с файлами — переименование, перемещение, копирование
- Просмотр структуры директорий
- Работа с файлами за пределами обычных инструментов Claude

**Пример запроса:** *«Покажи все файлы в папке mockups, отсортированные по дате»*

---

### 4. postgres
**Что делает:** SQL-запросы к PostgreSQL напрямую. SELECT, INSERT, UPDATE, DELETE. Просмотр схемы, таблиц, индексов. Подключён к БД `sklad` на localhost.

**Когда использовать:**
- Посмотреть данные в БД — «сколько товаров на складе?»
- Отладка — проверить что записалось после API-запроса
- Аналитика — сложные SELECT с JOIN и GROUP BY
- Проверить схему таблиц — какие колонки, типы, индексы
- Быстро поправить данные при разработке

**Когда НЕ использовать:**
- На продовой БД для UPDATE/DELETE без уверенности
- Для миграций — используй `schema.js`

**Пример запроса:** *«Покажи топ-10 товаров с наибольшим остатком на стеллажном складе»*

---

### 5. memory
**Что делает:** Граф знаний — хранит сущности (люди, проекты, концепции) и связи между ними. Данные переживают перезапуск сессии.

**Когда использовать:**
- Запомнить контекст проекта между сессиями
- Хранить информацию о людях, ролях, решениях
- Когда нужно чтобы Claude помнил что-то конкретное

**Пример запроса:** *«Запомни что деплой делается через PM2 и процесс называется c-site»*

---

### 6. puppeteer
**Что делает:** Браузерная автоматизация через headless Chrome. Открытие страниц, клики, ввод текста, скриншоты, выполнение JavaScript в консоли браузера, ожидание элементов.

**Когда использовать:**
- Быстрая проверка UI после деплоя — скриншот страницы
- Тестирование формы — заполнить и отправить
- Проверить что API отвечает через браузер
- Выполнить JS в контексте страницы
- Парсинг данных с веб-страниц

**Отличие от Playwright:** Puppeteer проще, только Chrome. Для базовых задач — достаточно.

**Пример запроса:** *«Зайди на страницу склада, авторизуйся и сделай скриншот дашборда»*

---

### 7. context7
**Что делает:** Загружает актуальную документацию библиотек прямо в контекст. Вместо того чтобы полагаться на устаревшие знания — подтягивает реальные доки.

**Когда использовать:**
- Работаешь с новым API React/Express/Tailwind — нужна свежая дока
- Claude не уверен в синтаксисе или API изменился
- Хочешь использовать новую фичу библиотеки
- Проверить актуальные параметры функции/хука

**Пример запроса:** *«Покажи документацию по React useOptimistic хуку из context7»*

---

### 8. fetch
**Что делает:** HTTP-запросы к любым URL. GET, POST и другие методы. Получает HTML/JSON, конвертирует в читаемый текст. Загрузка веб-страниц и API.

**Когда использовать:**
- Проверить что API сервера отвечает — `GET /api/products`
- Загрузить JSON из внешнего сервиса
- Прочитать веб-страницу и извлечь информацию
- Проверить статус сервера после деплоя
- Тестирование API эндпоинтов

**Пример запроса:** *«Проверь что API склада отвечает на /api/products/stats»*

---

### 9. playwright
**Что делает:** Продвинутая браузерная автоматизация от Microsoft. Chrome, Firefox, Safari. Навигация, клики, ввод, скриншоты, drag-and-drop, загрузка файлов, iframe, перехват сетевых запросов, эмуляция мобильных устройств.

**Когда использовать:**
- E2E тестирование — пройти по всем страницам и проверить
- Тестирование на мобильном разрешении
- Перехват и анализ сетевых запросов
- Сложные сценарии — drag-and-drop, загрузка файлов
- Нужна поддержка Firefox/Safari (puppeteer — только Chrome)

**Отличие от Puppeteer:** Мощнее, поддерживает несколько браузеров, лучше для тестов.

**Пример запроса:** *«Протестируй страницу инвентаризации — открой, создай задачу, проверь что она появилась»*

---

### 10. sqlite
**Что делает:** Работа с SQLite БД. SQL-запросы, создание таблиц, аналитика. Лёгкая локальная БД без сервера.

**Когда использовать:**
- Локальное прототипирование — набросать схему перед PostgreSQL
- Временное хранение данных при парсинге/импорте
- Аналитика по локальным данным
- Кэширование результатов

**Пример запроса:** *«Создай локальную SQLite БД и загрузи туда тестовые данные товаров»*

---

### 11. gdrive
**Что делает:** Доступ к Google Drive — чтение, создание, редактирование Google Docs/Sheets/Slides. Поиск файлов, управление папками, загрузка/выгрузка.

**Когда использовать:**
- Загрузить данные из Google Sheets (прайс-листы, каталоги)
- Создать отчёт в Google Docs
- Найти файл на Drive
- Экспорт данных из проекта в Google Sheets

**Требуется:** Настройка OAuth через Google Cloud Console при первом запуске.

**Пример запроса:** *«Загрузи прайс-лист из Google Sheets и сравни с данными в БД»*

---

### 12. sequential-thinking
**Что делает:** Пошаговый анализ сложных задач. Разбивает проблему на шаги, последовательно рассуждает, может пересматривать выводы. Глубокое логическое мышление.

**Когда использовать:**
- Архитектурные решения — «как лучше спроектировать модуль»
- Сложная отладка — «почему не работает, разбери по шагам»
- Планирование — «какие шаги нужны для внедрения фичи»
- Когда задача неоднозначна и нужно рассмотреть варианты

**Пример запроса:** *«Продумай архитектуру модуля аналитики с drill-down отчётами»*

---

### 13. everything
**Что делает:** Тестовый сервер от Anthropic. Демонстрирует все возможности MCP — ресурсы, инструменты, подсказки, подписки. Служит эталоном.

**Когда использовать:**
- Проверить что MCP-протокол работает
- Отладка подключения других серверов
- Изучение возможностей MCP

---

## Управление серверами

```bash
# Список и статус
claude mcp list

# Добавить сервер
claude mcp add <имя> -- <команда>

# Удалить сервер
claude mcp remove <имя>
```

## Серверы, требующие настройки

| Сервер | Что нужно |
|---|---|
| **gdrive** | OAuth credentials из Google Cloud Console |
| **postgres** | Строка подключения (сейчас: `localhost:5432/sklad`) |
| **slack** (не установлен) | `SLACK_BOT_TOKEN` + `SLACK_TEAM_ID` |
