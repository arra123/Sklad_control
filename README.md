## Быстрый старт

### 1. Установка зависимостей

```bash
cd backend && npm install
cd ../frontend && npm install
```

### 2. Настройка .env

Файл `backend/.env` содержит конфигурацию:
- **БД:** 5.42.100.180:5432, база `bd2`, таблицы с суффиксом `_s`
- **Порт:** 3017
- **JWT:** настроен
- **Логин по умолчанию:** admin / Admin12345

### 3. Запуск (разработка)

```bash
# Бэкенд
cd backend && node src/server.js

# Фронтенд (в отдельном терминале)
cd frontend && VITE_API_PROXY_TARGET=http://localhost:3017 npx vite --mode sklad --host 0.0.0.0 --port 5173
```

### 4. Деплой

Автоматический через GitHub Actions при пуше в `main`.
1. Checkout + Setup Node 20
2. `cd frontend && npm ci && npx vite build --mode sklad`
3. Очистка старых assets через SSH
4. SCP: backend/src + backend/package.json → сервер
5. SCP: frontend/dist → сервер
6. SSH: `npm install --production && pm2 restart`

Ручной перезапуск на сервере:
```bash
ssh root@147.45.97.155
cd /var/www/bem-dev.ru/sklad/backend
pm2 delete c-site; fuser -k 3017/tcp; sleep 2
pm2 start src/server.js --name c-site && pm2 save
```

## Роли и права

| Роль | Описание | Доступ |
|------|----------|--------|
| **admin** | Администратор | Полный доступ ко всем модулям |
| **manager** | Менеджер | Склады, задачи, аналитика, товары (без настроек и ролей) |
| **senior_storekeeper** | Старший кладовщик | Задачи, сотрудники, перемещения (по permissions) |
| **employee** | Сотрудник | Свои задачи, сканирование, перемещение, возвраты |

Права управляются через таблицу `roles_s` → поле `permissions` (JSON массив строк).
Middleware `requirePermission(...perms)` проверяет права пользователя.
Редактирование ролей и паролей: Сотрудники → вкладка Доступы.

## Ключевые фичи

### Склады
- **Стеллажный** (стеллажи → полки → коробки → товары)
- **Паллетный** (ряды → паллеты → коробки/товары)
- **Комбинированный** (оба типа в одном складе)
- **Коробочный** (склад только из коробок)
- 3 режима отображения: Список / Визуально / Карточки
- 3D-визуализация на Three.js с текстурными коробками, слоями, drill-down
- Цветовая палитра стеллажей для быстрой навигации (12 градиентных оттенков)
- Поиск товара на складе с отображением ячеек
- URL-параметры для шаринга (?view=cards, ?wh=20)
- Скрытие складов из статистики

### Коробки
- Многотоварные коробки (box_items_s, shelf_box_items_s)
- ШК с копированием, печатью, PDF
- Размерность коробки (box_size) в редакторе
- Быстрый перенос коробок между паллетами
- Перенос целой коробки на другой паллет

### Задачи и инвентаризация
- Типы задач: инвентаризация, оприходование, перенос с производства, сборка комплектов
- Задача на паллет: скан паллета → скан коробок по очереди
- Задача на полку: сканирование в коробки
- **Сборка комплектов:** забор компонентов → сборка → размещение (3 фазы)
- **Защита от дублей:** занятые полки/коробки серые и некликабельные при создании
- Пауза задачи (только admin) — блокировка сканирования, pause_log для статистики
- Инструкции в задачах (notes) — на старте и в процессе
- Звуковое оповещение на каждый скан
- Предупреждение при >50 шт. в коробке
- Умные подсказки при ошибках: кириллица, URL, дубль ШК, обрезанный ШК

### Перемещения
- 2 режима: поштучно и целой коробкой
- Экран выбора действия: Взять / Сдать / Между местами
- Админский режим «Переместить»
- Сканирование для подсчёта (каждый скан = +1 шт.)
- Блокировка сканирования во время перерыва
- Мульти-источник: товар помнит свой источник при мульти-сборке
- История перемещений на всех уровнях: полка, паллет, коробка

### Возвраты
- Сканирование баночки — система автоматически находит ячейку (коробку на полке)
- Интеграция с «Мой инвентарь» — кнопка «Положить назад»

### Заработок (GRAcoin)
- Автоначисление за каждый скан: инвентаризация, оприходование, сборка, перемещение
- Тарифы по типам задач (настраиваемые в Настройки → GRACoin)
- Виджет баланса в шапке сотрудника с анимацией +X GRA
- Страница заработка сотрудника: GRA за задачи, drill-down, ср. скорость
- Админская страница: сводка, лидеры, история по сотрудникам и задачам
- Фильтр по периоду: конкретный день или всё время
- Ручная корректировка баланса с аудит-логом
- Курс: 1 GRA = 0.01 ₽, ставка до 3 знаков после запятой

### Live-мониторинг
- Real-time мониторинг активности сотрудников (обновление каждые 5с)
- Activity Timeline: CSS-бар 5-мин бакетов (зелёный = работа, серый = простой)
- 3 типа перерывов: обед, отдых, технический — каждый свой цвет на таймлайне
- Технические паузы от админа с выбором интервала
- Date picker — просмотр активности за любой день
- Клик по задаче → popup с деталями, сканами, паузами
- Zoom/Pan таймлайна: +/−, стрелки, drag мышью
- TSD-компактный интерфейс для маленьких экранов (≤480px)
- 7 уровней цвета активности с настраиваемыми порогами

### Товары и сырьё
- Карточки товаров с расположением на складах
- Импорт каталога из МойСклад (API синхронизация)
- Системный штрихкод: назначение через звёздочку в карточке
- Комплекты с компонентами и тех. картами
- Редактирование тех. карт прямо в карточке товара (CRUD)
- Сырьё: группы (порошки, полуфабрикаты, расходники, этикетки, смеси)
- Навигация стеком: рецепт → ингредиент → карточка
- Проверка ШК через Ozon API (2 магазина) + WB API (2 магазина)

### Аналитика
- Сводка: задачи, сканы, ошибки, рейтинг сотрудников
- Инвентаризация: цветовые статусы (Свежий/Давно/Устарел/Не было)
- Аналитика V2: дерево складов + folder-style навигация
- Прогресс-бары по складам
- Drill-down в детали

### Управление пользователями
- Автосинхронизация с внешним сайтом: уволенные деактивируются
- Permission-based доступ (вместо hardcoded requireAdmin)
- Группировка по ролям с emoji-аватарками по должностям
- Копирование логин/пароль в один клик
- Поиск сотрудников без аккаунтов
- Жёсткая деактивация при синхронизации

### Обратная связь
- Кнопка фидбэка в сайдбаре админа и на странице задач
- Текст, голосовое сообщение (транскрипция), скриншот (Ctrl+V)
- Категории: баг, предложение, вопрос
- Админ-панель: фильтры, статусы, заметки

## API эндпоинты (14 файлов роутов)

### Auth
- `POST /api/auth/login` — логин (token + user с gra_balance)
- `GET /api/auth/me` — текущий пользователь
- `POST /api/auth/change-password` — смена пароля
- `POST /api/auth/fix-passwords` — rehash всех паролей из plain text

### Products
- `GET /api/products` — список (search, warehouse_id, placed_only, limit)
- `GET /api/products/:id` — детали с расположением на складах
- `POST/PUT/DELETE /api/products`
- `POST /api/products/check-ozon` — проверка ШК на Ozon
- `POST /api/products/check-wb` — проверка ШК на WB

### Materials (Сырьё)
- `GET /api/materials` — список сырья (с фильтрами по группам)
- `GET /api/materials/:id` — детали с рецептом
- `POST /api/materials` — создать материал
- `GET /api/materials/:id/recipe` — рецепт
- `GET /api/materials/:id/used-in` — где используется

### Warehouse (стеллажные склады)
- `GET /api/warehouse/warehouses` — список всех складов
- `CRUD /api/warehouse/racks` — стеллажи
- `CRUD /api/warehouse/shelves` — полки
- `CRUD /api/warehouse/shelf-boxes` — коробки на полках
- `GET /api/warehouse/visual-fbs/:id` — визуальное представление
- `GET /api/warehouse/movements` — история перемещений (shelf + movements merge)
- `GET /api/warehouse/box-movements` — история по коробке

### FBO (паллетные склады)
- `GET /api/fbo/warehouses` — паллетные склады
- `CRUD /api/fbo/rows` — ряды
- `CRUD /api/fbo/pallets` — паллеты
- `POST /api/fbo/pallets/:id/box` — создать коробку на паллете
- `POST /api/fbo/pallets/:id/item` — добавить товар напрямую
- `GET /api/fbo/visual/:id` — 3D визуальное представление

### Tasks
- `GET /api/tasks` — список задач (status, limit)
- `GET /api/tasks/busy-targets` — занятые цели
- `POST /api/tasks` — создать задачу (с проверкой занятости)

## Деплой и инфраструктура

### GitHub Actions (.github/workflows/deploy.yml)
При пуше в `main`:
1. Checkout + Setup Node 20
2. `cd frontend && npm ci && npx vite build --mode sklad`
3. Очистка старых assets через SSH
4. SCP: backend/src + backend/package.json → сервер
5. SCP: frontend/dist → сервер
6. SSH: `npm install --production && pm2 delete + start`

### Сервер
- **IP:** 147.45.97.155
- **PM2:** процесс `c-site`, порт 3017
- **Путь:** `/var/www/bem-dev.ru/sklad/`
- **nginx:** проксирует `/sklad` → `localhost:3017/sklad`
